Amway Online Corporate Directory Redesign

Intern Project

This was a project for all the IT interns at Amway. Our project was to redesign the current Online Corporate Directory (OCD). To do this we split into 3 teams: Interview team, Design team, and Development team. I was a member of both the Design and Development team. We worked using Agile processes in which we had many iterations of our design and prototype.

As a member of the Design team, I took the information gathered through the interviews and surveys conducted by the Interview team, and made an initial mockup of the new website. The old OCD was notorious for having too many field and small text, so we trimmed down the intitial fields to the most commonly used. We also created an 'advanced search' button to hide all the rarely used search fields. We changed the overall look of the OCD to make it more similar to the current Amway Intranet. We changed the size of the fields and buttons to make it easier to use. The emphasis on the color red changed to shades of blue. We also added a picture of the Amway Grand Plaza in downtown Grand Rapids to add to the Amway spirit and culture. I used Adobe Photoshop to make the many iterations of the mock-up.

As a member of the Development team, we implemented the design of the mock-up. My role was as a front-end developer. I mainly worked with HTML, CSS, and Bootstrap 3. Bootstrap helped to make our website responsive and mobile friendly. I would recieve basic html from the back-end developer and basically make it look pretty. Since we had only one virtual machine to run our code, we had to work in sessions, using lean techniques like partner programming.

Overall, I learned a lot about the Agile design process and about web development. Our prototype is currently being moved into production (which I'm really proud of) and will be taken over by Amway employees after the interns leave this summer.

Host a Website on GitHub

Beginners tutorial for GitHub

This is going to be a really short tutorial for all you people who have your html, css, JavaScript, etc. ready, but need a simple method to host your awesome website. Use GitHub.

Pros to using GitHub

Quick to setup
Free
Easy to make changes
Version Control
Custom error pages

Cons to using GitHub

Daunting for beginners
Only static pages (HTML, CSS, JS) - no PHP

Let's start.

First step is to get a GitHub account. Then you'll want to create a new repository and title it as such:

Obviously you will fill in your own user name instead of 'nnandigam23'. You can initialize the repository with a README file, if you want. Click "Create repository".

Now you should consider where you want your HTML, CSS, JS, etc. to be located. You will push these files from their location to Github. You can either have the files located locally on your computer, or store them on a cloud platform. For example, I keep all my web files on Dropbox. That way my website is not tied to my computer, instead it exists on the cloud. This could be useful if I were to lose my computer, or something happened to my hard drive. However, you are betting that Dropbox doesn't lose your files too. I recommend using a cloud-based storage application and keeping a recent copy of your work on a local machine as backup.

Using Git

Now we're going to use Git from the terminal to push your files to Github. Before that you'll want to copy the url of your repository to your clipboard. Other than copying the address from the top bar, there is a button under "HTTPS clone URL" on the right of the page that copies the url to your clipboard. So go ahead and launch your terminal (or powershell, if you're on Windows). Navigate to the folder where your web files are located. If you are new to the terminal, typing ls will display all the files and folders contained within the folder you are currently in. Typing cd Folder will get you to the folder specified (the folder has to be contained by the folder where you were previously). The command cd .. will get you up one folder. So once you're at the correct folder, enter the command:

git clone https://github.com/nnandigam23/nnandigam23.github.io

You'll want to insert that link that you copied where I have my url in the command. Now you should have a folder name 'username.github.io' where all your files are. In your Finder (or Windows Explorer), go ahead and move all those files into this new folder. Now in the terminal go to this new folder:

cd username.github.io

If you type in ls you should be able to see all the files you moved to this folder. So GitHub uses 'index.html' as your homepage, so you'll need to rename your main HTML page to 'index.html'. Once all your files are ready, enter:

git add *

This command readies all your changed files (in this case, new files) for deployment to GitHub. If you only want to add a single file, simply change the git add * to git add file.name. Next, you have to commit these changes. This is where you can type a message explaining the changes you have made during this particular upload. This is helpful when you are looking back on all the past versions of your code, and you want to know what changes were made:

git commit -m "blah blah blah"

You can enter your message in between the quotation marks. So now we can push our code to the repository. To do this, type:

git push origin master

Once you enter this, you should be prompted for your username and password. Enter these in, and you should be good to go! Wait for the success message to display, and go check your repository to see your newly-added files.

The moment you have been waiting for: viewing your website. To do this simply type in 'username.github.io' into your search box. Behold, your website. That's the address for your website, so go ahead and share it to everyone.

Making Changes to your Website

This is what makes GitHub ideal for implementing fixes. Simply edit your files in your username.github.io folder. Then in the terminal, once you've navigated to the folder, type:

git add .
git commit -m "your message"
git push origin master

And that's it. Hope you enjoyed. Happy coding

Rent Tap

This is a prototype UI that I designed for a Rent Sharing app that we were considering making. This was done during MHacks (the Hackathon at Michigan). I used Sketch 3 to make these workflows.

This app was designed to be a tool for roommates to easily organize paying for rent and other expenses. I made the layouts for 4 screens: the login page, the main profile page, the payment page, and the settings page. The profile page shows all the payments that must be made within "Apartment #304." The user can maintain several of these pages by simply swiping left or right. All the payment elements are in orange and all the charge elements are in blue.
